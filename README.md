**A from-scratch, high-performance C++ game engine. Open source, coming soon.**

Vanilla C++, or call it "C+-": C plus the good bits of C++, minus the bad. No
STL, no exceptions, quick compiles. One codebase spanning high-end desktop to
low-powered mobile. Multi-backend rendering. C#/.NET + Lua scripting. PBR/HDR.

## What is Ceili

Ceili is a from-scratch C++17 engine built on one founding rule: **no split between
the runtime and the tools.** The editor is the engine. One metadata system, one
table-based entity database, one serialization layer serve the game, the tools, the
network, and the undo stack alike.

- **The language, curated:** vanilla C++, "C+-": C plus the good bits of C++,
  minus the bad. No STL, no exceptions; handle-based, C-like public APIs over a
  C++ core; custom containers and arena allocators throughout. The payoff is
  quick compile times and code you can actually read.
- **Every device in between:** built to span high-end desktop to low-powered
  mobile from one codebase. Rendering is backend-agnostic behind one interface;
  D3D12 and Vulkan are the first two backends, with more (Metal among them) to
  follow.
- **Rendering:** physically based shading with image-based lighting; a fully
  data-driven HDR post-process chain (bloom, depth of field, lens flare,
  3D-LUT grading) with SDR / HDR10 / scRGB output.
- **Data layer:** columnar entity tables with compile-time read/write
  contracts driving a dependency-DAG scheduler; conflict-free systems run in
  parallel, off the main thread, automatically.
- **Scripting:** engine systems can be written in C++, C#, or Lua against the
  same interface, and hot-swapped live. The bindings are generated from the
  C-like public API, so all three languages track the engine automatically.
- **Multiplayer:** real-UDP transport, delta snapshots against acked baselines,
  client-side prediction with re-simulation from ack, and rollback. The same
  change-capture layer that replicates gameplay also powers live collaborative
  scene editing; every incumbent engine treats those as two unrelated
  codebases.
- **Ease of use, end to end:** from building the code (generate, build, run,
  quickly, with hot reload for materials, shaders, and scripts) to building
  worlds (entities, materials, and post chains authored as data that
  live-reloads in the editor). A from-scratch Language Server Protocol client
  gives the Studio editor go-to-definition and hover for Lua, C#, and inline
  HLSL; persistent per-run logs and on-crash minidumps make every failure
  self-explaining.
- **An embedded AI agent:** the engine hosts an in-process MCP server exposing
  its scripting API as tools, so an embedded Claude agent can drive the live
  engine. The tool catalogue is generated by the same generator that emits the
  language bindings.

## The numbers

The flagship stress test is 100,000 flocking boids, each a real entity with a
rendered surface, flowing through the full entity, data, scheduling, and
rendering stack, simulated and drawn live. Not a particle demo.

| Configuration                                              | Result    |
|------------------------------------------------------------|-----------|
| Fullscreen 4K game view, entire flock visible              | 85+ fps   |
| In-editor, 3D camera + 2D wireframe view simultaneously    | 55+ fps   |
| In-editor, typical editing case                            | 150+ fps  |

And because C++, C#, and Lua systems all run against the same data layer
through generated bindings, scripting stays close to native. On a 2048-agent
n-body benchmark (average ms per update):

| Implementation | Avg update |
|----------------|------------|
| C++            | 7 ms       |
| LuaJIT         | 8-10 ms    |
| C#             | 10-11 ms   |

The benchmark harnesses ship with the engine. Measure; do not take our word
for it.

## The story

Ceili is the direct successor to the Instinct engine (2002-2009). Instinct was
good: Sony Japan fast-tracked it onto their middleware program, NVIDIA used it
on stage to promote early CUDA, and Havok featured it in their promotional
material. Then a funding round collapsed in the wake of the 2008 financial
crisis, and it ended. The IP was acquired in the liquidation, and the engine
you are looking at was restarted from a blank page as its spiritual successor.

It is named Ceili (KAY-lee: https://en.wikipedia.org/wiki/C%C3%A8ilidh), after the author's granddaughter.
